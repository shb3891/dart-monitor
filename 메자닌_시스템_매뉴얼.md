## 1. 시스템 개요

### 1.1 이 시스템이 하는 일

113개+ 메자닌 채권(CB/EB/BW) 포트폴리오를 **자동으로 모니터링**하고, 중요한 이벤트가 발생하면 **실시간으로 텔레그램 알림**을 보내며, **언제든 텔레봇 채팅으로 종목 조회**할 수 있는 시스템.

### 1.2 핵심 기능 5가지

| 기능 | 설명 |
|---|---|
| **📊 자동 데이터 수집** | DART/SEIBRO API에서 매 시간 채권 정보 자동 갱신 |
| **🔔 실시간 알림** | 공시/이벤트/자본변동/주가 변동을 텔레그램으로 즉시 알림 |
| **📅 D-day 알림** | PUT/CALL/만기/리픽싱 등 6종 이벤트를 사전에 알림 (매일 09시) |
| **💬 봇 채팅** | 종목명/ISIN 입력으로 즉시 상세 정보 조회 (1-2초) |
| **💰 보유 추적** | 보유내역 xlsx 업로드 시 자동 매칭 + 변동 알림 |

### 1.3 구성 요소

```
1. dart-monitor (GitHub 레포): 데이터 수집
2. susung-alert (GitHub 레포): 알림 발송
3. Cloudflare Worker: 실시간 봇 응답
4. Google Sheet: 데이터 저장소
5. Telegram Bot ('수성_공시'): 알림 + 채팅 인터페이스
```

### 2.1 데이터 흐름

**① 정기 데이터 수집 (GitHub Actions)**
- 평일 장중 10분마다 / 장 전후 1시간마다
- DART/SEIBRO API → Google Sheet 자동 갱신

**② 알림 발송 (GitHub Actions)**
- 시트 데이터 + DART 실시간 확인 → 텔레봇으로 알림 발송

**③ 봇 채팅 응답 (Cloudflare Worker)**
- 사용자 메시지 → Worker 즉시 처리 → Google Sheet 조회 → 1-2초 내 응답

## 3. 텔레그램 봇 사용법

### 3.1 봇 정보

- **봇 이름:** 수성_공시
- **봇 채팅:** 팀 그룹 채팅
- **응답 시간:** 1-2초 (즉시)

### 3.2 명령어 전체 목록

#### 🔍 종목 검색 (그냥 텍스트)

| 입력 | 결과 |
|---|---|
| `천보` | 천보6CB 상세 정보 |
| `디앤디파마텍` | 디앤디파마텍2CB 상세 |
| `한세` | 한세엠케이6CB + 12CB (여러 매칭 시) |
| `KR6278282EB9` | ISIN으로 직접 검색 |

**응답 내용:**
- 종목명, ISIN, 발행/만기일
- 보유금액, 취득가, 평가액, 수익률
- Coupon, YTM, YTC, 행사가액, 리픽싱플로어
- 전환청구기간
- 다가오는 이벤트 (90일 내)
- 이후 PUT/CALL 다음 5건
- 발행사/교환대상 정보
- 자본변동 이력

#### 📊 보유 현황 명령어

| 명령어 | 설명 |
|---|---|
| `/help` | 도움말 표시 |
| `/list` | 보유 종목 전체 목록 (보유금액 큰 순) |
| `/total` | 총 보유 통계 (총액, 손익, 종류별) |
| `/upcoming` | 7일 내 다가오는 이벤트 |
| `/upcoming30` | 30일 내 다가오는 이벤트 |
| `/capital` | 최근 30일 자본변동 내역 |

#### 📌 매칭 관리 명령어 (관리자용)

| 명령어 | 설명 |
|---|---|
| `/match ISIN 주식코드 종목명` | 수동 매칭 등록 |
| `/status ISIN` | 매칭 정보 조회 |
| `/list_failed` | 매칭 실패 종목 |
| `/list_alias` | 별칭매칭 종목 |
| `/list_manual` | 수동확정 종목 |
| `/aliases` | 별칭사전 전체 |
| `/add_alias 원본 DART표기` | 별칭 추가 |

**예시:**
```
/match KR60156S1G12 435570 에르코스 농업회사법인
→ 에르코스에 대한 매칭 수동 등록

/add_alias 케이씨그린홀딩스 KC그린홀딩스
→ 별칭사전에 표기 차이 등록
```

### 3.3 사용 시나리오

#### 시나리오 A: 오늘 종목 체크
```
1. /total          → 전체 현황 빠르게 확인
2. /upcoming       → 이번 주 이벤트 체크
3. 천보            → 관심 종목 상세 확인
```

#### 시나리오 B: 신규 검토
```
1. 한국파마        → 신규 편입 종목 정보 확인
2. /capital        → 최근 자본변동 점검
```

#### 시나리오 C: 매칭 관리
```
1. /list_failed    → 매칭 실패 종목 확인
2. /match ...      → 수동 매칭 등록
3. /status ISIN    → 매칭 확인
```

## 5. 데이터 자동 갱신 흐름

### 5.1 main.py (가장 핵심)

**실행 주기:**
- 평일 장중 (09:00~15:30): 10분마다
- 평일 장 전후 (07~09시, 15:30~20시): 1시간마다
- 평일 밤/주말: 실행 안 함

**처리 내용:**
1. DART 전체 기업코드 로드 (4,000개)
2. 113개 종목 차례로 처리:
   - SEIBRO: 채권 기본 정보 + 행사가 + PUT/CALL
   - DART: 발행공시 파싱 (YTM, 리픽싱플로어, 전체 스케줄)
3. 포트폴리오 시트 갱신
4. 자본변동 감지 (최근 30일)
5. 풋콜스케줄 시트 빌드 (모든 차수)
6. 주식코드 시트 갱신

**소요 시간:** 약 22~30분 (113개 종목 × API 요청)

**워크플로우 파일:** `dart-monitor/.github/workflows/hourly_monitor.yml`

### 5.2 audit_check.py

**실행 주기:** 매일 KST 09:00
**처리:** 감사보고서 시트 갱신 (분기/연간 보고서 추적)
**워크플로우:** `dart-monitor/.github/workflows/daily_audit.yml`

### 5.3 alert.py

**실행 주기:**
- 평일 장중: 10분마다
- 평일 장 전후: 1시간마다

**처리:**
1. 보유 종목 신규 공시 체크 (DART)
2. 6종 이벤트 D-day 알림 (09시에만)
3. 신규 CB/EB/BW 발행결정 감지 (시장 전체)
4. 자본변동 즉시 알림 + 일일 요약(09시)
5. 주가 5% 변동 (장중)
6. 텔레그램 명령어 처리 (백업용)

**워크플로우:** `susung-alert/.github/workflows/run.yml`

### 5.4 update_holdings.py

**실행 주기:** `holdings/` 폴더에 xlsx push될 때마다 자동

**처리:**
1. holdings/ 폴더의 최신 xlsx 파일 인식
2. ISIN별 보유 정보 집계
3. 포트폴리오 시트와 매칭:
   - 매칭됨 → 보유 금액 갱신
   - 신규 ISIN → 시트에 추가 (노란색)
   - 사라진 ISIN → 회색 표시
4. 변동이력 시트 기록
5. 텔레봇 알림

**워크플로우:** `dart-monitor/.github/workflows/update_holdings.yml`

## 6. 보유내역 업데이트

### 6.1 정기 업데이트 방법 (월 1회 권장)

```
1. 새 보유내역 xlsx 파일 받기
   파일명 예시: 260730_보유내역.xlsx
   (날짜 형식은 자유, 가장 최근 수정 파일이 사용됨)

2. GitHub에 업로드:
   https://github.com/shb3891/dart-monitor/tree/main/holdings
   
3. [Add file] → [Upload files]

4. xlsx 파일 드래그앤드롭

5. 하단 [Commit changes] 클릭

6. 자동으로 워크플로우 실행됨 (push 트리거)
   → 약 3-5분 후 텔레봇으로 결과 알림 도착
```

### 6.2 보유내역 xlsx 필수 컬럼

| 컬럼명 | 필수 | 용도 |
|---|---|---|
| `종목코드` (ISIN) | ✅ | 매칭 키 |
| `종목명` | ✅ | 표시용 |
| `수량` | ✅ | 보유 금액 (원 단위) |
| `취득가` | 권장 | 손익 계산 |
| `시가평가액` | 권장 | 평가액 |
| `수익률` | 권장 | 수익률 표시 |

### 6.3 업데이트 후 일어나는 일

```
1. 텔레봇 알림: 신규 종목 (각각)
2. 텔레봇 알림: 매도/상환 종목 (각각)
3. 텔레봇 알림: 요약 (총 N개, M개 변동)
4. Google Sheet:
   - 포트폴리오: 보유금액 4개 컬럼 갱신
   - 보유내역_변동이력: 변동 사항 기록
   - 신규 종목: 노란색 표시
   - 사라진 종목: 회색 표시
```

### 6.4 ISIN 끝자리 'Z' 이슈

보유내역의 일부 ISIN은 마지막 자리가 'Z'로 되어 있을 수 있어요. 예:
- `KR6064261EAZ` ← 보유내역
- `KR6064261EA3` ← 실제 ISIN

이는 **전환 완료 후 콜만 남은 물량**을 별도 관리하는 것으로, 시스템은 별도 종목으로 처리합니다. 정상 작동.

## 7. 알림 시스템

### 7.1 알림 5종

#### 알림 1: 보유 종목 신규 공시
- **언제:** 보유 종목의 발행사 또는 교환대상 회사에 DART 공시가 올라오면
- **빈도:** 평일 매 10분 / 장외 1시간
- **포함:** 보고서명, DART 링크, 보유금액
- **예시:**
```
📢 보유종목 신규 공시
🔹 천보6CB
🏢 공시: 천보
📄 임원ㆍ주요주주특정증권등소유상황보고서
💰 보유: 33억원
🔗 DART 바로가기
```

#### 알림 2: 6종 이벤트 D-day
- **언제:** 매일 KST 09시
- **대상 이벤트:** 전환청구시작/종료, 만기, 리픽싱, 풋옵션, 콜옵션
- **알림 시점:**

| 이벤트 | 알림 |
|---|---|
| 전환청구시작 | D-7, D-1, D-day |
| 전환청구종료 | D-30, D-7, D-1 |
| 만기 | D-90, D-30, D-7, D-1 |
| 리픽싱 | D-7, D-1, D-day |
| 풋옵션 | D-7, D-1, D-day |
| 콜옵션 | D-7, D-1, D-day |

#### 알림 3: 신규 CB/EB/BW 발행결정
- **언제:** 평일 매 10분
- **대상:** DART 시장 전체에서 새 메자닌 발행 공시
- **용도:** 신규 투자 기회 발굴

#### 알림 4: 자본변동
- **언제:** 평일 매 10분 + 매일 09시 요약
- **대상 유형:** 주식병합, 액면병합, 주식분할, 액면분할, 무상증자, 유상증자, 감자, 주식배당
- **중요:** 행사가액 재계산 필요한 종목 추적!

#### 알림 5: 주가 5% 변동
- **언제:** 장중 (평일 09-15시)
- **대상:** 발행사 또는 교환대상 주식
- **기준:** 전일 대비 ±5% 이상

### 7.2 알림 받지 못할 때

체크 사항:
1. 봇이 그룹에 들어있는지
2. GitHub Actions 워크플로우가 정상 실행되었는지
   → https://github.com/shb3891/susung-alert/actions
3. 텔레그램 봇 토큰이 만료되지 않았는지

## 9. 트러블슈팅

### 9.1 GitHub Actions 실행 실패

**증상:** 워크플로우가 빨간색 ❌

**해결:**
1. Actions 페이지에서 실패한 실행 클릭
2. 로그 확인
3. 자주 발생하는 에러:

#### 에러 1: `ModuleNotFoundError: No module named 'xxx'`
```
원인: requirements.txt 누락 또는 패키지 미설치
해결: requirements.txt에 해당 패키지 추가
```

#### 에러 2: `RuntimeError: XXX 환경변수가 설정되지 않았습니다`
```
원인: GitHub Secrets 누락
해결: Settings → Secrets에 해당 키 추가
```

#### 에러 3: `gspread.exceptions.SpreadsheetNotFound`
```
원인 1: SHEET_ID 환경변수 누락
원인 2: Service Account에 시트 공유 권한 없음
해결: 
  1. Secrets의 SHEET_ID 확인
  2. Google Sheet 공유 목록에 Service Account 이메일 추가
```

#### 에러 4: Telegram Rate Limit (429)
```
원인: 짧은 시간에 너무 많은 메시지 발송
해결: 일시적 문제. 다음 실행에서 자동 해결
또는 코드의 time.sleep 늘리기
```

### 9.2 텔레봇 알림이 안 옴

**점검 순서:**
1. GitHub Actions 워크플로우 정상 실행되는지
2. TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID Secrets 확인
3. 봇이 그룹에 있는지
4. 봇 Group Privacy 설정 (BotFather → /mybots → Bot Settings → Group Privacy → Turn off)

### 9.3 봇 채팅 응답이 안 옴

**증상:** "/total" 입력했는데 응답 없음

**점검 순서:**
1. Cloudflare Worker 정상인지: https://dash.cloudflare.com/
2. Webhook 등록 확인:
```
https://api.telegram.org/bot<TOKEN>/getWebhookInfo
```
3. Worker Logs 확인 (Cloudflare 대시보드)
4. Worker Secrets 4개 모두 설정되어 있는지

**Webhook 재등록:**
```
https://api.telegram.org/bot<TOKEN>/setWebhook?url=<WORKER_URL>&secret_token=<SECRET>
```

### 9.4 보유내역 업로드 후 매칭 안 됨

**원인:**
- 보유내역의 종목명과 DART 등록명이 다름
- ISIN 끝자리 'Z' 이슈 (정상)

**해결:**
1. /list_failed 명령어로 매칭 실패 종목 확인
2. /match 명령어로 수동 매칭
3. /add_alias로 별칭 등록

### 9.5 데이터가 갱신 안 됨

**점검:**
1. Actions 실행 기록 확인
2. main.py 실행 시간 확인 (마지막 실행 언제?)
3. 시트의 "최근업데이트" 컬럼 확인
4. 수동 실행: Actions → Hourly Dart Monitor → Run workflow

### 9.6 자주 묻는 질문

#### Q: 신규 종목이 어떻게 자동으로 추가되나?
```
A: 보유내역 xlsx에 신규 ISIN이 있으면 update_holdings.py가 
   포트폴리오 시트에 자동 추가. 단, 회차/발행일 등은 비어있으므로
   다음 main.py 실행 시 SEIBRO에서 자동으로 채워짐.
```

#### Q: 자본변동이 감지되었는데 행사가가 그대로인데?
```
A: 자본변동 감지는 공시 발생 시점이고, 실제 행사가 재계산은 
   변동이 효력 발생한 후 SEIBRO 데이터에 반영됨.
   1~2주 정도 후 main.py 실행 시 자동 갱신.
```

#### Q: 매도/상환된 종목은 자동으로 사라지나?
```
A: 사라지지 않고 회색으로 표시됨. 시트에서 직접 삭제하거나 그대로 둬도 OK.
   (이력 보존 목적)
```

#### Q: 새 팀원 추가하려면?
```
A: 텔레그램 '수성_공시' 봇이 있는 그룹에 초대만 하면 됨.
   별도 권한 설정 불필요.
```

## 10. 시스템 확장 및 유지보수

### 10.1 새 알림 종류 추가

**예시:** 보유 종목의 PER이 특정 수치 이상일 때 알림

**작업 위치:** `susung-alert/alert.py`

**단계:**
1. 새 함수 추가 (예: `check_per_alert()`)
2. main 부분에 함수 호출 추가
3. 텔레봇 메시지 포맷 작성
4. 테스트

### 10.2 새 봇 명령어 추가

**작업 위치:** `Cloudflare Worker/worker.js`

**단계:**
1. 새 핸들러 함수 작성 (예: `handleNewCommand()`)
2. switch 문에 추가
3. `/help` 메시지 업데이트
4. Cloudflare 대시보드에서 코드 업데이트 + Deploy
5. 테스트

### 10.3 시트 컬럼 추가

**주의:** main.py가 정해진 컬럼 위치에 데이터를 씁니다. 컬럼을 중간에 끼우면 main.py 코드도 수정 필요.

**안전한 방법:** 시트 맨 끝(오른쪽)에 새 컬럼 추가.

### 10.4 main.py 실행 주기 변경

**파일:** `dart-monitor/.github/workflows/hourly_monitor.yml`

```yaml
on:
  schedule:
    - cron: '*/10 0-5 * * 1-5'   # ← 여기 수정
```

**참고:** GitHub Actions cron은 UTC 기준. KST = UTC + 9

### 10.5 보유 종목 100개 이상으로 늘어날 때

main.py 실행 시간이 늘어남 (50분 timeout 주의).
필요시 워크플로우의 `timeout-minutes` 늘리기:

```yaml
jobs:
  monitor:
    runs-on: ubuntu-latest
    timeout-minutes: 90   # ← 늘리기
```

## 11. 보안 주의사항

### 11.1 절대 노출되면 안 되는 정보

- **API 키들:** DART, SEIBRO, Telegram
- **GCP Service Account JSON**
- **WEBHOOK_SECRET**

이것들이 노출되면:
- 누구나 사용자 봇을 가장해서 메시지 발송 가능
- Google Sheet 접근 가능
- DART/SEIBRO API 한도 소진 가능

### 11.2 안전한 키 관리

#### ✅ 권장
- GitHub Secrets에만 저장
- Cloudflare Worker Secrets에만 저장
- 로컬 컴퓨터의 환경변수
- 비밀번호 매니저 (1Password, LastPass 등)

#### ❌ 금지
- 코드에 하드코딩
- Public 레포에 커밋
- 이메일/메신저로 평문 전송
- 스크린샷에 포함

### 11.3 키 노출 시 대응

만약 키가 노출되었다면 **즉시**:

1. **DART API 키 재발급:**
```
https://opendart.fss.or.kr → 로그인 → 인증키 관리 → 재발급
```

2. **SEIBRO 키 재발급:**
```
https://seibro.or.kr → 로그인 → API 관리 → 재발급
```

3. **Telegram Bot Token 재발급:**
```
@BotFather → /revoke → 봇 선택
```

4. **GCP Service Account 키 재발급:**
```
Google Cloud Console → IAM → 서비스 계정 → 키 추가/삭제
```

5. **GitHub Secrets + Cloudflare Secrets 모두 업데이트**
